# 固定 Dev 数据复核记录

复核完成于 2026-07-21。本次工作针对 E1/E2 暴露出的标签与决策边界问题进行定向人工复核，目标是恢复开发评测的可信度，不是扩充训练数据。

## 1. 复核动机

- E1 与 E2 在原始 Dev 上得到完全相同的汇总混淆矩阵，但有 20 张图片的 GOOD/BAD 决策不同、45 张完整 JSON 输出不同。
- 部分模型“错误”从图片观感上更像 Gold 标注问题，原始 Dev 指标可能低估模型能力并干扰 checkpoint 选择。
- 259 张 Test 也已发现标注误差，因此继续隔离，避免在标签未修正时产生误导性最终结论。

## 2. 复核范围与方式

- 原始 Dev 共 200 张：GOOD 149、BAD 51。
- 定向复核 57 张重点样本：E1/E2 共同错误 37 张、两模型决策分歧 20 张。
- 其余 143 张因两模型均与原 Gold 一致，本轮未逐张人工重标，继续保留原标签。
- 使用服务器本地复核网页逐张查看原图，并同时展示原 Gold、E1 和 E2 输出。
- 每张填写：原 Gold 是否正确、异常明显程度、最终 GOOD/BAD、主要类别和复核说明；结果实时保存为 JSON 和 CSV。
- 完成 57/57 后执行自动一致性检查；发现的 1 处填写矛盾重新查看并修正，最终 `consistency_issues=0`。

## 5. 对评测结果的影响

| 模型 | 标签口径 | Recall | FPR | Accuracy | F1 |
|---|---|---:|---:|---:|---:|
| E1 checkpoint-1248 | 原始 Dev | 50.98% | 14.77% | 76.50% | 52.53% |
| E2 checkpoint-1248 | 原始 Dev | 50.98% | 14.77% | 76.50% | 52.53% |
| E1 checkpoint-1248 | 条件修正版 | 67.24% | 6.34% | 86.00% | 73.58% |
| **E2 checkpoint-1248** | **条件修正版** | **74.14%** | **3.52%** | **90.00%** | **81.13%** |

复核表明，原始标签噪声既制造了 FN，也制造了 FP，并掩盖了 E2 相对 E1 的真实改善。修正后 E2 达到项目 Go 标准。

## 6. 使用边界与后续安排

- 修正版 Dev 只用于开发验证和 checkpoint 选择，禁止进入训练集。
- 本轮属于模型错误触发的定向复核，且复核时可看到模型输出，因此结果不能等同于全量、盲法、独立重标。
- 原始 Dev、复核记录、27 条变更清单和修正版 Dev 均独立保留并记录哈希，便于审计。
- E3 及后续实验统一使用修正版 Dev；最终结论仍由完成独立修正后的固定 Test 一次性验证。

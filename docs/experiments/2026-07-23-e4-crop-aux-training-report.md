# E4 局部辅助训练总结

实验：Qwen3.5-27B 多模态 LoRA；训练与固定 Dev 评测完成于 2026-07-23。E4 的目标是通过“完整图 + 异常/正常局部 crop”辅助监督，改善小手、五官、文字等细粒度异常的识别。当前已完成整图推理评测，局部复核推理尚未评测；固定 Test 全程隔离。

## 1. 数据集

- 沿用 broad-clean Train，共 8,026 张独立图片：GOOD 6,074、BAD 1,952；剔除 150 张唯一异常类别为“其他”的 BAD，混合 BAD 保留图片但删除“其他”实例。
- T1 整图样本沿用 BAD×2 采样，共 9,978 行：GOOD 6,074、BAD 3,904。
- Train 中共有 2,546 个有效异常 bbox；按每张 BAD 最多保留 2 个实例后得到 2,361 个异常 crop 候选。为构造 3,326 条 T2 局部 BAD，均匀重复 965 条，重复系数为 1.41。
- T3 使用 3,326 张不同的 GOOD 图片生成正常 crop，并与 T2 的宽高比例逐条匹配；最大比例误差低于 0.17%。
- 最终训练集共 16,630 行：T1 整图 9,978（60%）、T2 局部 BAD 3,326（20%）、T3 局部 GOOD 3,326（20%）。
- 小 bbox（面积不超过整图 1%）使用单个 `2.0×` crop；普通 bbox 使用 `1.5×`；大 bbox 自适应缩小扩张比例。每张 BAD 最多生成 2 个异常 crop。
- Dev 使用人工复核后的固定 200 张：GOOD 142、BAD 58，SHA256 为 `cd3a1e2d215b505526b7382a9ccf8d9acaca540e60dedf684cfaeca28cde3acb`；不回流训练。

## 2. 训练策略、参数与 Prompt

- 基座 Qwen3.5-27B，4×A100 80GB，BF16，DeepSpeed ZeRO-2，FlashAttention 2。
- 与 E2 相同：LLM+aligner 使用 `all-linear` LoRA，ViT 冻结；rank 16、alpha 32、dropout 0.05。
- 学习率 5e-5，cosine scheduler，warmup ratio 0.05，weight decay 0.1；全局 batch 16。
- `max_length=3072`，图像 token 上限 1,024；训练 2 epoch，共 2,080 step。
- 每 260 step 验证并保存，共得到 8 个 checkpoint。adapter 含 996 个张量，其中 aligner 4 个、ViT 0 个，训练范围符合设计。

T1 保持原整图 Prompt：

```text
<image>
检查这张图片。输出decision、categories和reasons。decision只能是GOOD或BAD。
```

T2/T3 使用双图局部 Prompt：

```text
<image>
<image>
第一张图片是完整图，仅用于理解上下文。
第二张图片是待检查的局部区域。
只判断第二张局部区域是否存在明显生成异常。
输出decision、categories和reasons。decision只能是GOOD或BAD。
```

该设计用完整图提供位置和结构上下文，但要求模型只判断第二张 crop，避免完整图其他异常污染局部标签。图片级 `decision` 仍是核心目标，`categories` 和短 `reasons` 仅作辅助监督。

## 3. 结果统计和分析

### 训练运行

- 完成 2,080/2,080 step，耗时约 4 小时；显存稳定在 60.05 GiB，无 OOM、NaN 或异常退出。
- 最终 `train_loss=0.1462`；最终 Dev `eval_loss=0.2472`、`eval_token_acc=0.9463`。
- trainer 按 token loss 选择 checkpoint-780（`eval_loss=0.2044`）。780 之后 Dev loss 持续升高，存在过拟合信号。

### 固定 Dev 整图二分类

8 个 checkpoint 均采用与 E2 相同的 non-thinking 确定性生成协议，仅输入完整图；payload JSON 和完整 schema 合法率均为 100%。

| Step | Recall | FPR | Accuracy | F1 |

| **260** | **60.34%** | 11.27% | 80.50% | 64.22% |
| 520 | 55.17% | 8.45% | 81.00% | 62.75% |
| 780 | 48.28% | 7.75% | 79.50% | 57.73% |
| 1040 | 31.03% | 0.70% | 79.50% | 46.75% |
| 1300 | 56.90% | 4.93% | 84.00% | 67.35% |

按 Recall 优先规则，整图模式选择 checkpoint-260：TP 35、FN 23、FP 16、TN 126。它未达到 Recall≥72% 的 Go 标准。

| 模型 | TP | FN | FP | TN | Recall | FPR | Accuracy | F1 |
|---|---:|---:|---:|---:|---:|---:|---:|---:|
| **E2 checkpoint-1248** | **43** | **15** | **5** | **137** | **74.14%** | **3.52%** | **90.00%** | **81.13%** |
| E4 checkpoint-260 | 35 | 23 | 16 | 126 | 60.34% | 11.27% | 80.50% | 64.22% |
| E4 checkpoint-1560（最高 Accuracy/F1） | 34 | 24 | 7 | 135 | 58.62% | 4.93% | 84.50% | 68.69% |

因此，E4 在“仅输入整图”的模式下明显弱于 E2，不能替代当前 E2 champion。最低 token loss 的 checkpoint-780 也不是图片级最佳点，再次说明 checkpoint 必须按生成式二分类指标选择。

## 4. 归因

1. **局部辅助任务与整图任务存在负迁移。** E4 每个 epoch 仍完整看过全部 T1，因此退化不能简单归因于整图样本训练不足；新增 40% 双图局部任务更可能改变了共享 LLM+aligner adapter 的决策边界。
2. **输入与监督分布更复杂。** 同一 adapter 同时学习单图整图判断和双图局部判断，且局部样本强调小结构，可能削弱整图模式下对全局关系和正常边界的稳定判断。
3. **局部 BAD 存在适度重复采样。** 2,361 个候选扩展为 3,326 行，重复系数 1.41，可能进一步强化局部异常敏感性；checkpoint-260 的 FP 增至 16，表现出一定误报代价。
4. **继续训练没有修复整图能力。** 后半程 FPR 降低，但 Recall 始终未恢复到 E2 水平；2 epoch 增加了训练充分度，但没有带来整图指标提升。
5. **当前结论只覆盖整图推理。** E4 专门训练的“双图局部检查”能力尚未进入评测，因此只能判定“E4 整图模式失败”，不能据此否定完整的局部复核方案。

## 5. 下一步计划

1. E2 checkpoint-1248 继续作为当前最佳整图模型；E4 暂不进入 Test。
2. 优先评测 E4 checkpoint-260 的“整图 + 2×2 局部复核”模式：已有整图结果直接复用，仅补充四个局部结果。
3. 在固定 Dev 上统计局部复核新增 TP 与新增 FP，并评估简单 OR、局部多票等融合规则；所有规则只在 Dev 选择。
4. 若局部复核无法至少达到 Recall≥72%，停止当前 E4 路线；若召回显著提升但 FPR 上升，再调整融合规则，而不是立即重训。
5. 固定 Test 在数据修正、模型 checkpoint、局部划分和融合规则全部锁定后，只运行一次最终评测。
